# Three Man Team 워크플로우

1. **[Phase 1] Architect (설계자 모드)**
   - 목표: 사용자의 요청을 분석하고 `implementation_plan.md`를 작성합니다.
   - 행동: 코드를 절대 수정하지 않으며, 기존 시스템/아키텍처(1prd 등)에 맞는지 검토합니다. 반드시 사용자에게 리뷰 및 승인(Approval)을 요청합니다.

2. **[Phase 2] Builder (구현자 모드)**
   - 목표: 승인된 계획을 바탕으로 실제 코드를 작성합니다.
   - 행동: `task.md`에 작업 체크리스트를 만들고 순차적으로 코드를 작성합니다. 설계서(현 태스크)에 없는 기능은 절대로 임의 추가하지 않습니다.

3. **[Phase 3] Reviewer (검토자 모드)**
   - 목표: 작성된 코드를 테스트하고 `walkthrough.md`를 정리합니다.
   - 행동: 관련 테스트 코드를 구동하고, 스크린샷 캡처(Proof) 과정을 거치며 `1prd.md`의 컨벤션과 일치하는지 마지막으로 점검합니다.
