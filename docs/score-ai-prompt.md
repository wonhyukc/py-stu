# AI(Gemini) 활용 학습 노력도 평가 기준 (5점 만점)

본 문서는 학생들이 제출한 AI(Gemini) 대화 기록을 바탕으로, 단순 문제 풀이를 넘어선 **주도적 학습 노력**을 평가하기 위한 채점 기준(Rubric) 및 프롬프트 가이드입니다.

## 📝 평가 목적
* 학생이 단순히 정답을 얻기 위해 AI를 사용했는지, 아니면 학습의 도구로써 깊이 있게 활용했는지 판별합니다.
* '왜(Why)'와 '어떻게(How)'를 묻는 주도적인 태도와 꼬리 질문(Follow-up questions)에 가장 높은 가산점을 부여합니다.

---

## 📊 5점 만점 채점 기준 (Rubric)

### **1~2점 (노력 미흡 / 단순 복붙)**
* 시험이나 과제 문제를 그대로 복사하여 붙여넣기만 하고 대화가 종료된 경우.
* AI의 답변을 읽고 추가적인 질문이나 상호작용이 전혀 없는 수동적인 태도.
* (예시) *문제 붙여넣기 -> AI 정답 도출 -> 끝*

### **3점 (기본 / 단순 추가 질문)**
* 문제를 복사/붙여넣기 했으나, 답변 중 이해가 안 가는 짧은 단어나 개념에 대해 **한 번 정도 단순 추가 질문**을 한 경우.
* (예시) *"이 단어가 무슨 뜻이야?", "이 코드만 다시 보여줘."*

### **4점 (우수 / 학습 노력 보임)**
* 문제의 정답을 얻는 데 그치지 않고, **"왜 그런지(Why)"** 이유를 묻거나 관련 개념에 대해 추가적인 설명을 요구한 경우.
* AI의 설명을 바탕으로 **다른 예시를 보여달라고 요청**하며 개념을 이해하려고 시도한 경우.
* (예시) *"왜 옵션 A는 안 되는 거야?", "마진 병합에 대한 다른 예시 코드를 짜줘."*

### **5점 (최우수 / 주도적 학습 및 응용)**
* 본인이 먼저 코드를 작성해보고 AI에게 **리뷰나 디버깅을 요청**한 경우.
* 배운 개념을 본인의 프로젝트나 다른 상황에 확장 적용하여 **심도 있는 꼬리 질문**을 여러 번 이어나간 경우.
* (예시) *"이 개념을 내 포트폴리오의 이력서 부분에 적용하고 싶은데 코드를 어떻게 수정하면 좋을까?", "방금 알려준 방법 말고 서버 사이드에서 방어하는 코드를 직접 짜봤는데 한 번 봐줄래?"*

## 🤖 채점용 AI 프롬프트 예시 (Score Prompt)
자동 채점 봇(Bot)을 구성할 때 아래의 프롬프트를 활용할 수 있습니다.

```text
당신은 웹 프로그래밍 과목의 채점 조교입니다.
주어진 학생과 AI의 대화 기록을 읽고, 학생의 '학습 노력도'를 1점부터 5점까지의 척도로 평가하세요.

[평가 기준]
- 1~2점: 질문만 단순 복사/붙여넣기 한 경우
- 3점: 단순한 추가 질문이 1회 정도 있는 경우
- 4점: 정답의 원리나 이유를 묻고 추가 예시를 요구하는 등 학습하려는 의지가 돋보이는 경우
- 5점: 배운 내용을 자신의 상황(프로젝트 등)에 응용하려 하거나, 본인의 코드를 디버깅 요청하며 깊이 있는 꼬리 질문을 이어간 경우

출력 형식:
점수: [1~5 숫자]
평가 사유: [왜 이 점수를 부여했는지 2~3문장으로 요약]
```
