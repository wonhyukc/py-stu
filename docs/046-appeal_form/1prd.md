# 이의제기 설문지 (Appeal Form) 요구사항 정의

## 1. 개요
평가 결과에 대해 학생들이 이의를 제기할 수 있도록 체계적인 설문지를 작성한다.
- **언어**: 한/영 병용 표기

## 2. 설문지 필수 항목
- **학번**: 제출자의 학번 (식별용). 정규표현식을 통해 `10자리 2026으로 시작`하는지 검증
- **이름**: 학번을 기반으로 자동 채우기
- **트랙**: 소속 트랙 (471 7xx, 7xx 등). 학번을 기반으로 자동 채우기
- **유형**: 이의제기 유형 (예: 과제, 상호평가, 시험, 이메일과제, 수업참여 등). 드롭다운 선택
- **받았어야 한다는 점수**: 학생 본인이 생각하는 정당한 점수. 숫자만 입력 가능하도록 검증
- **문항 이의제기 내용**: 구체적인 이의제기 사유 및 내용 (텍스트 입력)

## 3. 기술적 구현 방안 (사전 채우기 링크 활용)
- 구글 폼 기본 기능으로는 학번 입력 시 이름/트랙 실시간 자동완성이 불가능하므로, 파이썬 스크립트를 활용하여 각 학생별로 **학번, 이름, 트랙이 미리 채워진(Pre-filled URL) 고유 링크**를 일괄 생성하여 배포하는 방식으로 구현한다.
- **학생별 고유 링크(개인화된 URL) 개념**: 모든 학생은 동일한 접속 링크를 쓰는 것이 아니라, 각자의 고유한 파라미터가 포함된 개별 링크를 부여받게 됩니다. 즉, 70명의 학생이 있다면 서로 다른 70개의 URL이 생성되며, 학생이 본인의 링크를 클릭하면 처음부터 자신의 데이터(학번, 이름, 트랙)가 완벽하게 타이핑되어 있는 상태의 폼을 마주하게 되어 오탈자로 인한 데이터 오류를 100% 방지할 수 있습니다.
