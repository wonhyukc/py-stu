# 주제별 과제 제출 상호평가 기획 문서 (Issue #20)

## 1. 목적 (Objective)
- 주제별 과제(예: py05 등 비코딩/개념 과제)의 특성을 반영하여 학생들이 서로의 과제를 공정하고 효과적으로 평가할 수 있는 상호평가(Peer Evaluation) 기준과 프로세스 구축
- 주관적 평가를 배제하고, 학생들이 루브릭 표의 문항 하나하나를 직접 확인하면서 개별적으로 점수를 입력하고 처리할 수 있는 **문항별 확정 채점 시스템(방법론)** 설계

## 2. 평가 기준 및 방식 확정 

### 2.1 루브릭 기반 개별 문항 체크 시스템
각 조건(Condition)마다 명확한 배점을 할당하고 평가자가 항목별로 구글 폼에서 단답형으로 평가 체크.
*py05 루브릭 기반 측정 항목:*
1. **제출 기한 제한**: 해시(Hash) 커밋 시각 확인 (3점/0점)
2. **접근성(Public)**: 링크 정상 접속 여부 (1점/0점)
3. **세부 조건 충족**: 문서 내 목표 개수 등 충족 여부 (1점/0점)
4. **증빙 확인**: 지정된 인터랙션 횟수 확인 (1점/0점)

### 2.2 심사 수집 시스템: 구글 폼(Google Forms) 결합
- **장점**: 직관적인 UI 제공 및 무료, 입력 데이터가 기존 구글 시트에 자동 적재되어 중앙화.
- **방식**: 학생은 폼에 접속하여 Q1, Q2, Q3 등 분할된 질문에 배점(선택지)만 선택. 피드백 주관 개입 원천 차단.

### 2.3 학생 간 매칭 정책 (Matching) 및 활용 기술
- **배정 인원**: 학생 1인당 동료 과제 **3명** 배정
- **사용 기술**: **구글 앱스 스크립트 (GAS)** 
- **방식**: 제출 마감 후 구글 시트에 내장된 스크립트(GAS)를 실행하면, 편중 없이 순환(Circular) 배정으로 서로를 크로스 체킹할 수 있는 매칭 결과 시트가 즉시 생성됩니다.
