# Issue #20: 상호평가 시스템 (Python 엔진 + Google Sheet DB 수정안)

## 1. 목적 (수정됨)
- 유지보수가 어려운 구글 앱스 스크립트(GAS) 로직과 매번 이메일을 발송해야 하는 번거로움을 모두 제거합니다.
- 학생들은 익숙한 구글 시트와 양식(구글 폼)을 통해 과제와 평가를 확인 및 제출하며, 뒷단의 핵심 배점 처리 및 분석 매칭 비즈니스 로직은 철저하게 안전한 로컬 Python 스크립트가 실행합니다.

## 2. Python 기반 워크플로우

### [Phase 1: Python 오프라인 매칭 및 시트 업로드]
1. **데이터 수합 및 조인**:
   - `input/students/*.md` 파일을 파라미터로 하여 수강생 **전체 마스터 목록**을 읽습니다 (미제출자 포함 전원이 "평가자"로서 참여).
   - 구글 시트 API와 `secret.json`을 사용하여 과제를 제출한 링크(Google Sheets)에 접속, **제출자 목록**을 추출합니다 (이들만이 "피평가자"가 됩니다).
2. **비대칭 랜덤 풀 매칭**:
   - 분반별로 제출 여부를 확인한 뒤, 미제출자라 하더라도 모두 평가자로서 3건을 평가하도록 배정합니다.
   - 피평가자(제출자) 수가 적을 경우, 한 피평가자가 여러 번 중복 평가를 받을 수 있도록(최소 3명 이상에게 피드백을 받도록) 피평가자 풀을 균등 순환 배정하는 로직을 적용합니다. (자기 자신 평가는 룰에 따라 제외)
3. **공지용 결과 산출 (메일 ❌)**: 
   - 스크립트가 분반과 이름/학번이 분리된 CSV 규격의 데이터를 `output/peer-eval-result.csv` 형태로 저장합니다.
   - 교수님이 이를 복사(Copy)해서 구글 시트의 "배정 결과" 탭에 그대로 붙여넣습니다.

### [Phase 2: 단일 구글 폼 수합 및 파이썬 채점 스캔]
1. **통합 폼으로 깔때기 수합**: 학생들은 자신의 피평가자를 확인한 뒤, 이전에 기획 완료한 '통합 1개짜리 상호평가 폼'에 들어가 채점 데이터를 보냅니다.
    - **통합 폼 시트 링크(상호평가 포함)**: [Link](https://docs.google.com/spreadsheets/d/1_o-F6UaQ2WOe0nH2zuT_0xpwiOm1ebmWo2sEQzQptEk/edit?resourcekey=&gid=427896056#gid=427896056&fvid=8905371)
2. **파이썬 평가 검증 엔진 (`check_evaluations.py`)**: 
   - 파이썬이 이번엔 해당 평가 폼 시트를 읽어 들여와 최종 점수를 산출합니다.
   - **이상치 판독(Warning System)**: 루브릭 규격 외의 점수나 경고를 받아야 할 (1점 이상) 이상 채점이 있을 경우 파이썬 터미널 로그에 노란색 ⚠️ 알림을 띄워 시각적으로 필터링해 줍니다.
   - **상호평가 신뢰도 기반 점수 부여 (다수결 원칙)**: 동일한 피평가자(제출자)에 대해 3명의 평가자가 내린 점수 중, "다수결(Majority Vote)"로 최종 점수를 확정합니다. 이때, 다수결의 결과와 다르게 잘못 채점한 (즉, 소수의견에 해당하는) 평가자는 본인의 '평가 참여 점수'를 0점 처리하며, 올바르게 채점한 다수의 평가자에게는 1점을 부여합니다.
3. **최종 병합**: 이 결과는 기존의 `check_assignments.py` 마이크로 채점 시스템과 합쳐져 관리됩니다.

### [Phase 3: 평가 결과 가중치 병합 및 레포팅]
1. **타겟 구글 시트 자동 동기화**: [응답 통합 구글 시트](https://docs.google.com/spreadsheets/d/166MzQg-W6r9GEynt1bOlr8hUex0Rvx6Yky2ffSVNtKM)를 자동으로 스캔합니다.
2. **최종 획득 점수 산출 로직 체계화**:
   - `제출 점수 (가중치 0.8)`: 본인이 과제를 제출하여 다수결 평가를 통해 산정된 점수 반영
   - `평가 점수 (가중치 0.2)`: 동료의 과제를 1건 이상 올바르게 채점 시 획득되는 점수. 단, 학생 수가 적어 채점자 배정이 3명 미만으로 할당되었을 때는 불이익이 없도록 자동 스케일링합니다. (예: 2명 평가 배정 시 건당 1.5점 배분으로 총합 3점 만점 달성 보장)
   - 두 항목 합산 시 **매주 1점 만점**을 획득하도록 결과값을 정규화(Normalize)합니다.
3. **분석 결과 파일 출력**: 
   - CLI 실행 시 주차, 강좌 번호 인자(`--course py --week 06`)를 받아 채점 후, `output/py-06-peer-score.csv` 형식의 이름 규칙으로 자동 분류 저장합니다.
   - 교수님이 쉽게 검증할 수 있도록 **동일 제출자(평가 대상자) 학번 기준 오름차순**으로 자동 정렬합니다.
