# 과제 이메일 자동 채점 정책 (Score Email Policy)

이 문서는 Gmail을 통해 수집된 학생 과제 제출 메일을 파싱하고 점수를 부여하는 기준을 정의합니다.

## 1. 메일 수집 범위 및 필터링 정책
*   **수집 기간**: 지난 주 월요일 `09:00`부터 현재 주차 월요일(또는 마감일) `09:00`까지로 한정합니다.
    *   *기술적 조치*: Gmail API `after` 쿼리가 날짜까지만 지원하므로, 파이썬 코드 레벨에서 `datetime`을 통해 `09:00`를 정밀하게 잘라냅니다.
*   **제외 대상 (Blacklist)**:
    1.  제목에 `"was edited"` 문자열이 포함된 구글 문서 수정 알림 메일
    2.  발신자가 `comments-noreply@docs.google.com` 인 구글 문서 댓글 알림 메일

## 2. 채점(Scoring) 규칙
이번 주차 과제(예: `0.6`)에 대해 다음 기준을 적용하여 CSV 파일로 저장합니다. 

*   **정상 처리건 (2점)**:
    *   정해진 수집 기간(기한 내) 도착함.
    *   제목이 `과제 0.6 [10자리 학번]` 또는 `assignment 0.6 (10자리 학번)` 과 같이 양식을 완벽히 준수해야 함 (**학번 필수**).
    *   **첨부파일이 없어야 함**.
*   **규칙 위반 (1점)**:
    *   해당 주차 과제로 추정되거나 이메일 제출 의도가 명확하나, 다음 중 하나라도 위반한 경우:
        *   제목 양식 오류 (학번 누락, 띄어쓰기, 주차 표시 오류 등)
        *   첨부파일이 포함된 경우
*   **나머지 내용 / 예외 처리건 (0점)**:
    *   **학번으로 해당 학생을 찾을 수 없는 경우 (무조건 0점 처리)**.
    *   제출한 내용이 완전히 다른 주차 과제(예: `과제 0.5`)에 해당하나, 제출 이메일로 분류되어 로직상 걸러진 경우.
    *   시스템에서 `점수: 0` 으로 부여하고 수동으로 원인을 확인합니다.
    *   *(주의) 순수 개인 이메일이나 학교 스팸 메일이 모두 0점으로 CSV에 포함되는 것을 방지하기 위해, 과제 제출 목적의 메일(과제, assignment, 학번 등 키워드 포함)로 국한하여 가져오거나, 명백한 스팸은 파이썬 로직에서 필터링 하도록 합니다.*

## 3. 실행 파이프라인
위 기준을 적용하여 메일을 새롭게 Fetch하고, 결과를 `output/mail06.csv` (또는 해당 주차 csv)에 업데이트합니다.
